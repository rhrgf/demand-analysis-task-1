# 「需求分析」课前作业

## 总结流程

1. 等待第一轮选课时段
2. 进行第一轮权重选课
    1. 分别进入课程对应的A, B, C平台
    2. 在课程目录中选择自己要选的课（也可以在课表预览中单击区域进行选课）
    3. 输入权值并确定
    4. 系统确认选课
3. 公布第一轮选课结果
3. 等待第二轮选课时段
4. 进行第二轮抢课选课
    1. 分别进入课程对应的A, B, C平台
    2. 在课程目录中选择自己要选的课（也可以在课表预览中单击区域进行选课）
    3. 确定选课
    4. 系统确认结果
5. 公布第二轮选课结果
5. 等嗲第三轮选课时段
6. 进行第三轮抢课选课
    1. 分别进入课程对应的A, B, C平台
    2. 在课程目录中选择自己要选的课（也可以在课表预览中单击区域进行选课）
    3. 确定选课
    4. 系统确认结果
7. 公布第三轮选课结果

## 寻找痛点

1. 虽然已经实行分段选课，但是在第一轮选课的开始和结束和抢课开始会出现选课高峰，服务器无法承载这么大的访问量，会导致同学选课时出现问题
2. 抢课的决定性因素是网速和电脑配置，这有失公允，使得所有人起点不同
3. 分时段抢课系统有bug，可以进入没有开放的选课页面
4. 抢课的时候重新按照所有步骤选课太花时间
2. *选课时虽然可以看到自己的已修学分但是看不到还差多少，不方便选课

### 解决方案

1. \#
2. 改进抢课算法，使得大家起点相同
3. 修改bug
4. 可以在上一轮选课的时候增加备抢选项
